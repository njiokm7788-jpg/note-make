# UI Designer 分析报告

## 角色视角

作为 UI 设计师，我从视觉层次、交互反馈、组件设计和用户认知负担角度分析当前前端的优化机会。

## 当前界面分析

### 1. 上传区域 (ImageUploader)

**现状问题**:
- 两个独立的上传区域视觉上分离，用户需要理解"原图"和"标注图"的区别
- 缺少粘贴板支持，用户需要额外的拖拽/点击操作
- 上传成功后的视觉反馈较弱（仅边框颜色变化）

**优化建议**:

#### A. 粘贴板支持设计
```
┌─────────────────────────────────────┐
│  📋 支持 Ctrl+V 粘贴图片            │
│                                     │
│  ┌─────────┐    ┌─────────┐        │
│  │ 原始图片 │ → │ AI标注图 │        │
│  │ (空/已选)│    │ (空/已选)│        │
│  └─────────┘    └─────────┘        │
│                                     │
│  粘贴顺序：先原图，后标注图          │
└─────────────────────────────────────┘
```

- 全局监听粘贴事件
- 智能填充：第一次粘贴填充原图，第二次填充标注图
- 粘贴时显示 toast 提示确认

#### B. 视觉状态增强
- **空状态**: 虚线边框 + 淡色背景 + 上传图标
- **悬停状态**: 边框高亮 + 背景加深 + 放大图标动画
- **已选状态**: 实线边框 + 绿色标记 + 缩略图 + 文件名
- **处理中状态**: 脉冲动画 + 进度指示

### 2. 参数设置面板 (ProcessingPanel)

**现状问题**:
- 参数过多，普通用户难以理解（文字检测阈值、遮罩膨胀半径等）
- 滑块缺少直观的效果预览
- 没有预设方案，每次都需要手动调整

**优化建议**:

#### A. 预设方案卡片设计
```
┌─────────────────────────────────────┐
│  快速预设                           │
│                                     │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐  │
│  │ 标准 │ │ 轻量 │ │ 强调 │ │ 自定 │  │
│  │ ✓   │ │     │ │     │ │ ⚙️  │  │
│  └─────┘ └─────┘ └─────┘ └─────┘  │
│                                     │
│  ▼ 高级设置（可折叠）               │
└─────────────────────────────────────┘
```

**预设方案定义**:
- **标准模式**: 平衡的默认参数，适合大多数场景
- **轻量标注**: 低透明度，标注不遮挡原文
- **强调标注**: 高透明度，标注更醒目
- **自定义**: 展开高级参数面板

#### B. 参数分组与简化
- **基础参数**（默认展示）: 预设选择、标注透明度
- **高级参数**（折叠）: 文字检测阈值、遮罩膨胀、边缘引导等

### 3. 预览区域 (ImagePreview)

**现状问题**:
- 四宫格布局信息密度高，普通用户可能只关心最终结果
- 中间步骤（提取的标注层）对普通用户意义不大
- 放大查看功能发现性低

**优化建议**:

#### A. 简化预览布局
```
默认视图（普通用户）:
┌─────────────────────────────────────┐
│           最终结果（大图）           │
│                                     │
│     [点击查看对比] [下载结果]        │
└─────────────────────────────────────┘

对比视图（点击展开）:
┌─────────────────────────────────────┐
│  原图  │  标注图  │  最终结果        │
└─────────────────────────────────────┘
```

#### B. 交互增强
- 默认只显示最终结果大图
- 提供"查看对比"按钮展开详细视图
- 支持左右滑动对比原图和结果
- 放大按钮更明显，支持双击放大

### 4. 整体布局优化

**现状**: 左右分栏，左侧上传+设置，右侧预览

**优化建议**:
```
优化后的流程布局:

步骤 1: 上传图片
┌─────────────────────────────────────┐
│  [原图] ────────────→ [标注图]      │
│  支持拖拽、点击、粘贴               │
└─────────────────────────────────────┘
          ↓
步骤 2: 选择预设
┌─────────────────────────────────────┐
│  [标准] [轻量] [强调] [自定义]      │
└─────────────────────────────────────┘
          ↓
步骤 3: 查看结果
┌─────────────────────────────────────┐
│         最终效果预览                │
│     [查看对比] [下载结果]           │
└─────────────────────────────────────┘
```

## 视觉设计建议

### 色彩系统
- **主色调**: 保持现有的 primary 蓝色系
- **成功状态**: 绿色 (#22c55e)
- **预设卡片**: 使用不同的强调色区分
- **高级设置**: 使用紫色系保持一致

### 动效设计
- **上传成功**: 轻微弹跳 + 绿色闪烁
- **预设切换**: 平滑过渡 (300ms)
- **预览更新**: 淡入淡出 (200ms)
- **按钮点击**: 轻微缩放 (scale 0.98)

### 响应式考虑
- 移动端：垂直堆叠布局
- 平板：保持两栏但调整比例
- 桌面：当前布局优化

## 优先级排序

1. **P0 - 粘贴板支持**: 显著减少操作步骤
2. **P0 - 预设方案**: 降低认知负担
3. **P1 - 预览简化**: 聚焦核心结果
4. **P2 - 视觉状态增强**: 提升反馈质量
5. **P2 - 参数折叠**: 渐进披露高级功能
