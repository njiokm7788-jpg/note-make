# 前端优化指导规范

## 项目背景

**项目名称**: 笔记图片叠加工具
**项目目标**: 提取 AI 标注并与原图合成
**当前状态**: 功能完整，需要优化用户体验

## 优化目标

**核心目标**: 优化前端，符合人类操作习惯
**目标用户**: 普通用户（需要简单直观的操作流程）

## 确认的优化方向

### 1. 上传体验优化

**用户选择**: 支持粘贴板直接粘贴图片

**具体要求**:
- 支持 Ctrl+V / Cmd+V 直接粘贴图片
- 粘贴后自动识别并填充到对应的上传区域
- 保留现有的拖拽和点击上传功能
- 提供清晰的粘贴操作提示

### 2. 参数设置优化

**用户选择**: 预设方案模式

**具体要求**:
- 提供常用场景的预设方案（如：标准模式、高对比度、轻标注等）
- 一键应用预设方案
- 保留高级参数调整入口（可折叠）
- 预设方案应覆盖大多数使用场景

### 3. 预览体验优化

**优化重点**:
- 优化预览和结果展示方式
- 提升视觉反馈的即时性
- 简化预览区域的信息层次

## 优化原则

1. **简单优先**: 默认展示最简单的操作路径
2. **渐进披露**: 高级功能按需展开
3. **即时反馈**: 操作后立即给予视觉反馈
4. **容错设计**: 允许用户轻松撤销和重试

## 技术约束

- 基于现有 React + TypeScript 架构
- 使用 Tailwind CSS 进行样式设计
- 保持与现有组件的兼容性
- 不引入新的外部依赖（除非必要）

## 分析角色

1. **ui-designer**: 从视觉设计和交互反馈角度分析
2. **product-manager**: 从用户需求和功能优先级角度分析
