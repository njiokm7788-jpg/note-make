## 2. Space Utilization Metrics

### Vertical Space Breakdown (1080px viewport)

| 区域 | 高度 | 占比 | 问题 |
|-----|------|------|------|
| Header | 64px | 6% | OK |
| Main padding-top | 32px | 3% | 可压缩 |
| 上传卡片 | ~420px | 39% | **过高** |
| 间距 gap-6 | 24px | 2% | 可压缩 |
| 设置面板 | ~450px | 42% | **过高** |
| Footer | 56px | 5% | OK |
| Main padding-bottom | 32px | 3% | 可压缩 |
| **Total** | **1078px** | 100% | 超出视口 |

### Problem Diagnosis

**Issue 1: 上传区垂直堆叠**
- 两个 ImageUploader 垂直排列，各占 ~180px
- 包含 padding 和标题后达到 ~420px
- **根因**: 设计假设用户需要大面积拖拽区

**Issue 2: ProcessingPanel 默认展开过多**
- 快速预设 3x3 grid: ~120px
- 高级设置折叠按钮: ~48px
- 高级参数展开时: ~200px
- 实时预览开关: ~48px
- 操作按钮: ~100px
- **根因**: 所有选项平铺展示，缺乏优先级

**Issue 3: 预览区空状态浪费**
- 空状态占用 ~300px 高度
- 仅显示图标和提示文字
- **根因**: 设计追求视觉平衡而非空间效率

## 3. Scrolling Behavior Analysis

### Current Scroll Requirements

**1080p 显示器 (常见笔记本)**:
- 首屏可见: Header + 上传区部分
- 需要滚动: 设置面板下半部分 + 操作按钮
- **滚动距离**: ~200-300px

**1440p 显示器 (外接显示器)**:
- 首屏可见: 大部分内容
- 需要滚动: 高级设置展开时
- **滚动距离**: ~100px

**768px 高度 (小屏笔记本)**:
- 首屏可见: 仅 Header + 上传区
- 需要滚动: 整个设置面板
- **滚动距离**: ~400px

### User Journey Pain Points

```
用户操作流程:
1. 上传原图 ──────────────────────────────────────────┐
2. 上传标注图 ────────────────────────────────────────┤ 首屏
3. 选择预设 ──────────────────────────────────────────┘
4. [可选] 调整高级参数 ───────────────────────────────┐
5. 查看预览 ──────────────────────────────────────────┤ 需滚动
6. 下载结果 ──────────────────────────────────────────┘
```

**核心问题**: 步骤 3-6 需要滚动，打断操作连贯性

## 4. Component-Level Issues

### ImageUploader
- 拖拽区 p-6 (24px padding) 过大
- 预览图 max-h-40 (160px) 占用过多
- 空状态图标 w-12 h-12 可压缩

### ProcessingPanel
- 卡片 p-6 (24px) 可改为 p-4
- 预设按钮 p-3 可改为 p-2
- 高级设置默认折叠是正确的，但展开后内容过多

### ImagePreview
- max-h-80 (320px) 限制合理
- 但空状态 py-16 (64px) 过大

## 5. Responsive Breakpoint Analysis

### Current Breakpoints
- `lg:grid-cols-3`: 1024px+
- 默认: 单列堆叠

### Issues at Different Widths

**1024-1280px (lg)**:
- 左栏 1/3 宽度 (~340px) 足够
- 右栏 2/3 宽度 (~680px) 预览充足
- **问题**: 垂直空间不足

**768-1024px (md)**:
- 单列布局，所有内容垂直堆叠
- **问题**: 滚动距离更长

**< 768px (sm)**:
- 移动端布局
- **问题**: 需要大量滚动
